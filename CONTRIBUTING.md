# Contributing

## Getting Started

Create a focused branch for each change and keep pull requests small enough to review easily.

## Development Notes

- Do not commit local virtual environments, model weights, caches, or experiment outputs.
- Prefer clear, reproducible scripts over one-off local steps.
- Update documentation when behavior or setup changes.

## Pull Requests

- Summarize what changed and why.
- Mention how the change was tested.
- Call out any follow-up work or known limitations.
